# 운동 기록 앱 — Phase 1 / Slice 2.5 구현 명세

> **이 문서를 읽는 AI 코드 에이전트에게**: 이 문서는 Slice 2가 완료된 상태에서 진행하는 **작은 개선 슬라이스(2.5)** 명세서다. 5일간 실사용 후 발견된 일상 사용성 개선 사항만 처리한다. 새 화면이나 큰 기능은 없고, 기존 기능에 작은 추가/개선이다. Slice 3(통증/부상 입력)은 별도 슬라이스. **반드시 "범위 외" 섹션을 먼저 확인하라.**

---

## 1. 슬라이스 컨텍스트

**전체 그림에서의 위치**:
- Phase 1 (메모 앱 대체) / Slice 2.5
- Slice 1: 기본 기록/조회/수정 (✅ 완료)
- Slice 2: 루틴, 백업, RPE/RIR, 컨디션 (✅ 완료, 5일 실사용)
- **Slice 2.5: 일상 사용성 개선 (이 문서)**
- Slice 3: 통증/부상 입력 (다음, 미정)

**Slice 2 완료 상태에서 시작**:
- 종목 라이브러리, 운동 세션, 히스토리, 설정 화면 모두 동작
- JSON 백업/복원 동작
- 루틴 생성/사용 동작
- RPE/RIR 입력 동작
- 단위 전환(kg/lb), 저장공간 표시 동작
- Vercel 배포 + PWA 설치 완료 (`vercel.json` 포함)

**Slice 2.5의 목적**:
실사용 5일 후 발견된 3가지 일상 불편을 해소:
1. 같은 무게/횟수 반복 입력 비효율
2. 시드 데이터에 없는 종목 추가 불가
3. 액션 버튼 색상이 모두 같아 빠른 인지 어려움

## 2. 이 슬라이스의 범위

### 만들 것

**기능 1: 같은 세션 내 이전 세트 복사**
- 세트 추가 시 "⟲ 이전 세트와 동일" 버튼 제공
- 직전 세트의 값을 새 세트에 복사
- 본세트/워밍업 구분에 따라 복사 동작 다름

**기능 2: 커스텀 종목 추가 UI**
- 종목 목록 화면 우상단에 "+" 버튼
- 필수 4개 + 선택 입력으로 종목 추가
- 자동 추정 + 기본값으로 입력 부담 최소화

**기능 3: 색상 시스템 적용**
- 의미별 색상 일관 적용 (위험=빨강, 진행=초록, 정보=파랑)
- 다크 모드 친화적 채도
- 한 화면 3가지 색 이하 원칙

**부가 기능**:
- Exercise에 `muscle_mapping_confidence` 필드 추가 ('verified' | 'user_estimated')
- 커스텀 종목 수정/삭제 (수정은 사용자 추가 종목만)
- 사용된 커스텀 종목 삭제 시 soft delete (목록 숨김, 데이터 보존)

### 범위 외 (의도적으로 안 만드는 것)

- ❌ 통증 로그 입력 (Slice 3)
- ❌ 무통증 체크인 (Slice 3)
- ❌ 부상 프로필 등록 (Slice 3)
- ❌ 백업 권장 알림 카드 — **v4.6 spec에 명시했지만 Slice 2.5에선 안 만듦** (Slice 3 또는 별도로)
- ❌ 자동 변형 추정 (커스텀 종목에) (Phase 3 이후)
- ❌ joint_stress 자동 추정 정교화 (Phase 3 이후)
- ❌ 추천 엔진 (Phase 2)
- ❌ 전반적 디자인 시스템 리뉴얼 — 색상 컨벤션만 적용, 디자인 토큰화는 안 함

> **중요**: 작은 슬라이스이므로 1~2일 내 완료가 목표. 디자인 시스템화나 큰 리팩토링은 하지 않는다.

## 4. 데이터 모델 변경

### Exercise 테이블에 필드 1개 추가

```typescript
{
  // ... 기존 필드들
  muscle_mapping_confidence: 'verified' | 'user_estimated';  // v4.6 신규
}
```

**마이그레이션**:
- 기존 시드 종목: 모두 `'verified'`
- 기존 커스텀 종목 (있다면): `'user_estimated'`
- Dexie 스키마 인덱스 변경 없음 → version bump 불필요

**Dexie 마이그레이션 패턴**:
```typescript
// schema.ts
db.version(1).stores({ /* 기존 그대로 */ });

// 첫 진입 시 기존 데이터에 필드 추가 (idempotent)
async function ensureMuscleMappingConfidence() {
  const exercises = await db.exercises.toArray();
  const toUpdate = exercises.filter(e => !e.muscle_mapping_confidence);
  if (toUpdate.length === 0) return;

  await db.transaction('rw', db.exercises, async () => {
    for (const ex of toUpdate) {
      await db.exercises.update(ex.id, {
        muscle_mapping_confidence: ex.is_custom ? 'user_estimated' : 'verified'
      });
    }
  });
}
```

`ensureSeeded()` 호출 후 `ensureMuscleMappingConfidence()` 호출.

## 5. 기능 명세

### 5.1 같은 세션 내 이전 세트 복사

**위치**: 운동 세션 화면, 각 종목 블록의 "세트 추가" 버튼 옆

**UI**:
- "세트 추가" 버튼 (기존, 초록색)
- 새 버튼: "⟲ 이전 세트와 동일" 또는 ⟲ 아이콘 버튼 (회색 또는 파랑)
- 기본 동작은 변경 없음 — 빈 입력으로 세트 추가

**복사 동작 규칙**:

| 직전 세트 | 추가할 세트 | 복사 항목 |
|-----------|------------|----------|
| 본세트 | 본세트 | 무게, 횟수, RPE/RIR |
| 워밍업 | 워밍업 | 무게, 횟수 (RPE/RIR 미복사) |
| 워밍업 | 본세트 | 버튼 비활성화 |
| (없음) | 첫 세트 | 버튼 비활성화 |

**구현 포인트**:
- 새 세트가 워밍업인지 본세트인지는 사용자가 추가 시 선택 (기존 동작 유지)
- 복사 후에도 사용자가 값 수정 가능 (예: 횟수만 변경)
- 복사된 세트도 자동 저장 흐름 동일

**예시 시나리오**:
1. 세트 1 (본세트): 80kg × 5회, RPE 7 입력
2. "⟲" 버튼 탭 → 세트 2 (본세트): 80kg × 5회, RPE 7 자동 입력
3. 횟수만 8회로 수정 → 자동 저장

### 5.2 커스텀 종목 추가 UI

**진입점**: 종목 목록 화면 우상단에 "+ 종목 추가" 버튼

**모달 또는 화면 입력 필드**:

**필수**:
1. **종목 이름** (자유 텍스트, 1~30자)
2. **카테고리**: 컴파운드 / 아이솔레이션 (라디오)
3. **기구**: barbell / dumbbell / machine / cable / bodyweight / ez_bar (드롭다운)
4. **주동근**: 모든 Muscle을 그룹별로 보여주는 드롭다운

**선택**:
5. **보조근 1** (선택): Muscle 드롭다운 + 자극 계수 (0.5 또는 0.25 라디오)
6. **보조근 2** (선택): 동일 형식
7. **권장 휴식 시간** (선택): 기본값 자동 표시 (수정 가능)
   - 컴파운드: 180초
   - 아이솔레이션: 90초
8. **권장 반복 범위** (선택): 기본값 자동 표시 (수정 가능)
   - 컴파운드: [6, 12]
   - 아이솔레이션: [10, 15]

**자동 추정/기본값 (사용자에게 안 물어봄)**:
- `weight_increment_kg`: 컴파운드 2.5, 아이솔레이션 1.25
- `fatigue_factor`: 1.0
- `recommendation_bias`: 1.0
- `joint_stress_region_ids`: 빈 배열 (Phase 3에서 보강)
- `movement_patterns`: 빈 배열
- `alternative_group_id`: null
- `skill_level`: 'medium'
- `stability`: 'moderate'
- `unilateral`: false

**저장 시 동작**:
- 새 Exercise 생성 (`id`는 UUID 또는 `custom_<timestamp>` 형식)
- `is_custom: true`
- `muscle_mapping_confidence: 'user_estimated'`
- ExerciseMuscleMapping 레코드들 함께 생성 (트랜잭션)
- 종목 목록에 즉시 반영

**검증**:
- 종목 이름이 기존 종목과 중복되면 경고 (저장은 가능, 확인 다이얼로그)
- 주동근은 반드시 1개
- 보조근에 주동근과 같은 근육 선택 시 거부

### 5.3 커스텀 종목 수정/삭제

**수정**:
- 커스텀 종목(`is_custom: true`)에만 수정 버튼 표시
- 시드 종목은 수정 버튼 비활성화 또는 미표시
- 수정 화면은 추가 화면과 동일, 기존 값 채워진 상태

**삭제**:
- 커스텀 종목(`is_custom: true`)에만 삭제 버튼 표시
- 사용 이력 없으면 hard delete (Exercise + ExerciseMuscleMapping 삭제)
- 사용 이력 있으면 (SessionExercise에 참조됨) **soft delete**:
  - Exercise에 `is_archived: true` 필드 추가 (boolean, 기본 false) 
  - 종목 목록에서 숨김
  - 과거 세션 데이터는 그대로 표시 가능
  - 검색 결과에 안 나옴
  - 별도 화면("보관된 종목")에서 복원 가능 — 이건 v4.7에서 추후 결정

> **Note**: `is_archived` 필드 추가는 v4.6 spec에 명시되지 않은 추가 결정사항이다. v4.7-changes.md에 메모.

### 5.4 색상 시스템 적용

**의미별 색상 컨벤션** (Tailwind 클래스):

| 의미 | 클래스 | 사용처 |
|------|--------|--------|
| 진행/추가/시작 | `bg-emerald-400` text-white | 세트 추가, 종목 추가, 세션 시작, 루틴 시작 |
| 위험/종료/삭제 | `bg-red-400` text-white | 세션 종료, 세트/종목 삭제, 루틴 삭제, 데이터 초기화 |
| 정보/기본/저장 | `bg-sky-400` text-white | 저장, 확인, 백업 다운로드 |
| 경고/주의 | `bg-amber-400` text-black | 경고 메시지, 미저장 변경 표시 |
| 비활성/취소/보조 | `bg-gray-500` text-white | 취소, 비활성화 상태, 보조 액션 |

**적용 범위**:
- 모든 버튼
- 액션이 명확한 카드/링크
- 알림 또는 토스트 메시지

**적용 안 하는 곳**:
- 본문 텍스트 (흰색 유지)
- 카드 배경 (회색 계열 유지)
- 일반 입력 필드 (테두리만 회색)

**다크 모드 친화적 채도**:
- `400` 단계 사용 (다크 배경에 부드러움)
- 호버 시 `500`으로 약간 진하게
- `bg-red-500` 같은 더 진한 색은 위급 상황에만 (예: 영구 데이터 삭제)

**한 화면 3가지 색 이하**:
- 운동 세션 화면 예시:
  - 초록 (세트/종목 추가)
  - 빨강 (세션 종료, 삭제)
  - 회색 (취소, 보조)
- 세 가지 + 흑백 텍스트 = 충분

## 6. 검증 시나리오

**시나리오 A: 이전 세트 복사 (본세트)**
1. 새 세션 시작, 벤치프레스 추가

**시나리오 B: 이전 세트 복사 (워밍업 → 본세트 비활성화)**
1. 새 세션, 종목 추가
2. 세트 1 (워밍업): 40kg, 8회

4. ⟲ 버튼이 비활성화되어 있거나 클릭 시 안내 ("워밍업 세트는 본세트로 복사할 수 없습니다")

**시나리오 C: 커스텀 종목 추가**
1. 종목 목록 → "+ 종목 추가" 탭
2. 이름: "케이블 풀오버", 카테고리: 아이솔레이션, 기구: cable
3. 주동근: 광배 (lats), 보조근: 삼두 (0.5)
4. 저장
5. 종목 목록에 "케이블 풀오버" 표시됨
6. 검색 "풀오버" 시 노출됨
7. 새 세션에 추가 가능

**시나리오 D: 커스텀 종목 수정**
1. 종목 목록에서 위에서 만든 "케이블 풀오버" 탭
2. 수정 버튼 → 보조근에 0.25 자극으로 가슴 추가
3. 저장 → 변경 반영

**시나리오 E: 커스텀 종목 soft delete**
1. "케이블 풀오버"를 한 세션에서 사용
2. 종목 목록 → 케이블 풀오버 수정 화면 → 삭제 버튼
3. 안내: "사용 이력이 있어 보관 처리됩니다. 과거 세션은 유지됩니다."
4. 종목 목록에서 사라짐
5. 히스토리에서 해당 세션을 보면 "케이블 풀오버" 표시 유지

**시나리오 F: 시드 종목은 수정/삭제 불가**
1. "바벨 벤치프레스" 탭
2. 수정/삭제 버튼이 보이지 않거나 비활성화

**시나리오 G: 색상 컨벤션 확인**
1. 운동 세션 화면 진입
2. "세트 추가" 초록색 확인
3. "종목 추가" 초록색 확인
4. "세션 종료" 빨강색 확인
5. 세트 삭제 버튼 빨강색 확인
6. 색상 명확히 구분되어 보임

**시나리오 H: 마이그레이션**
1. Slice 2까지 사용한 기존 데이터로 앱 진입
2. 모든 기존 시드 종목이 `muscle_mapping_confidence: 'verified'`로 표시
3. 기존 동작 변화 없음

## 7. 작업 가이드

### 작업 순서 권장

1. **데이터 마이그레이션**: Exercise에 `muscle_mapping_confidence` 추가 + 기존 데이터 마이그레이션
2. **색상 컨벤션 적용**: 가장 작은 변화부터 — 세션 화면 버튼 3개 색 변경
3. **다른 화면 색상 일관 적용**: 히스토리, 루틴, 설정 화면 버튼 색
4. **이전 세트 복사 버튼**: 세트 입력 UI에 ⟲ 버튼 추가, 복사 로직
5. **커스텀 종목 추가 화면**: 종목 목록 → "+" 버튼 → 추가 모달/화면
6. **커스텀 종목 수정/삭제**: 사용 이력 확인 → hard delete 또는 soft delete
7. **검증 시나리오 A~H 수동 테스트**

### 보고 시점
