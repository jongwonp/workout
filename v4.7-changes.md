# v4.7 변경사항 메모 (Slice 2.5 작업 중/후)

> Slice 2.5 구현 중 내려진 결정과, 실사용하며 추가로 발견될 마찰을 모아두는 문서.
> 충분히 쌓이면 spec v4.7로 정리한다. (slice2.5-spec.md §9 기준)

## A. Slice 2.5에서 확정한 추가 결정 (spec v4.6에 명시 안 됨)

### A1. `Exercise.is_archived` 필드 추가 (soft delete)
- 사용 이력이 있는 커스텀 종목 삭제 시 hard delete 대신 `is_archived: true`.
- 종목 목록/검색/picker에서는 숨기고(`getAllExercisesWithPrimaryMuscle`에서 필터), 과거 세션은 id 직접 조회라 이름이 그대로 표시됨.
- **보관된 종목 복원 UI는 미구현.** 필요해지면 별도 화면(설정 등)에서 `is_archived=false` 토글. v4.7에서 결정.

### A2. Exercise 타입 확장 위치
- `seeds.ts`는 "수정 금지" 원칙이라 seed의 `Exercise`를 건드리지 않고, `src/types/index.ts`에서 `extends SeedExercise`로 신규 필드(`muscle_mapping_confidence`, `is_archived`)를 확장.
- seed 데이터 삽입 시 `seed-loader`가 기본값(`'verified'`, `false`)을 채움.
- 기존(Slice 2) 데이터는 `ensureExerciseDefaults()` idempotent 마이그레이션으로 보정.

### A3. 이전 세트 복사(⟲) 동작 해석
- 단일 "⟲ 이전과 동일" 버튼은 **직전 세트의 워밍업 여부를 그대로 따른다.**
  - 본세트 복사 → 무게/횟수/RPE·RIR
  - 워밍업 복사 → 무게/횟수만 (강도 미복사)
- spec 표의 "워밍업→본세트 비활성화"는 별도 분기 대신, 복사가 직전 세트 타입을 미러링하므로 워밍업 값이 본세트로 들어가는 경우 자체가 발생하지 않도록 한 것. 본세트가 필요하면 "+ 세트 추가"(빈 세트) 사용.
- 첫 세트(직전 세트 없음)에서는 버튼 비활성화.
- **재검토 포인트**: 실사용 중 "워밍업 뒤에 본세트를 복사 톤으로 빠르게 넣고 싶다"는 니즈가 생기면 버튼 분리 검토.

### A4. 커스텀 종목 추가/수정 UI 형태
- 모달이 아니라 별도 라우트 화면(`/exercises/new`, `/exercises/:id/edit`)으로 구현. 기존 페이지 기반 네비게이션과 일관.
- 진입점: 종목 목록 헤더 "+ 종목 추가"(초록) + 커스텀 종목 카드 우상단 ✎ 버튼.
- 카테고리 변경 시 휴식/반복 기본값 자동 갱신은 **사용자가 해당 필드를 직접 건드리지 않았을 때만** 적용(`touchedRest` 플래그).

### A5. 보조근 매핑 role
- 커스텀 종목의 보조근은 자극계수(0.5/0.25)와 무관하게 `role: 'secondary'`로 저장(주동근만 'primary', coefficient 1.0).
- 시드의 `mapping()` 헬퍼는 계수로 role을 추론(0.25→'minor')하지만, 커스텀 입력 UX가 "보조근"으로 단일화돼 있어 일관되게 'secondary' 사용.

## B. 색상 시스템 적용 메모 (라이트 테마)
- 실제 앱은 라이트 테마(흰 배경)라 spec의 다크 전제 `400` 단계 대신 **`600`(hover/active `700`) + `text-white`** 사용.
- 의미-색 매핑: 진행/추가/시작=emerald, 위험/종료/삭제=red, 정보/저장/완료=sky, 경고=amber, 보조/취소=gray(테두리).
- 적용 버튼: 세션(종목추가·세트추가=emerald, 세션종료=red, 수정완료=sky), 세션시작/컨디션 시작=emerald, 루틴 추가=emerald/저장=sky, 백업=sky, 종목 추가=emerald/삭제=red.
- **재검토 포인트**: 한 화면 3색 이하 원칙이 깨지는 화면이 있는지 실사용 후 점검(특히 설정 화면).

## D. 추후 발견 사항 (실사용하며 누적)

### D1. 횟수 −/+ 스텝 버튼 추가 (이전 세트 복사 후속 개선)
- 이전 세트 미러링 후 횟수만 바꾸려고 입력칸 탭→키보드 전환→삭제→재입력하는 번거로움 해소.
- `SetInput`의 횟수 입력을 `[ − ] [입력칸] [ + ]` 스텝퍼로 변경(`adjustReps`, 최소 1 클램프, 직접 타이핑도 유지). 무게는 기존 빠른 증감 버튼 유지.
- 워밍업 세트는 컨테이너가 `bg-amber-50`라 배경 없는 버튼이 노랗게 비치는 문제 → 스텝 버튼에 `bg-white` 명시. (무게 빠른증감 버튼엔 이미 적용돼 있었음)

### D2. 세션 중 종목 완료 → 접힘(포커스) 기능
- **배경**: 세션 중 이미 끝낸 종목이 계속 펼쳐져 있어, 지금 하는 종목에 집중하기 어려움.
- **동작(확정안)**: 종목 블록 **하단**의 "✓ 이 종목 완료" 버튼 → 해당 종목이 한 줄 요약(`✓ 종목명` + `대표세트×횟수 · N세트`)으로 **제자리 접힘 + 흐리게(opacity-60)**. 요약 행 탭하면 다시 펼쳐 편집 가능(이때 하단에 "▴ 접기"·"완료 해제" 노출). 대표세트 = 본세트 중 최대 무게.
- **버튼 위치(UX 결정)**: 완료 버튼은 헤더가 아니라 **하단**(세트 다 하고 → 바로 아래 완료 → 다음 종목 흐름). 단 하단의 초록 채움 `+ 세트 추가`와 의미 혼동을 막기 위해 완료는 **초록 테두리**(border-emerald-600 text-emerald-700) + 구분선(border-t)으로 격을 다르게 둠. 헤더엔 완료 표시로 종목명 앞 `✓`만.
- **저장**: `SessionExercise.is_done`(boolean) DB 영구 저장. 생성 시 false, 토글 시 갱신. 기존 데이터는 `undefined→false` 취급(읽기 시 coalesce)이라 별도 마이그레이션 없음. 펼침(재오픈) 상태는 화면 임시(WorkoutSessionPage 로컬 Set).
- **이번에 제외(선택했다 뺀 것)**: 헤더 진행률(2/5), 완료 종목 아래로 정렬, 잠금(read-only). 필요하면 후속.
- **재검토 포인트**: 완료 종목이 많아지면 "아래로 정렬"이나 진행률이 더 유용해질 수 있음. is_done을 히스토리/추천(완료율, 중도 포기 패턴)에 활용 여지.

### D3. SessionExercise 필드 추가도 타입 확장이 아니라 직접 정의
- `SessionExercise`는 seeds가 아니라 `src/types/index.ts`의 사용자 데이터 타입이라 `is_done`을 인터페이스에 직접 추가(= seeds 확장 패턴 불필요). seed 파생 타입(Exercise 등)만 확장 패턴 적용.
