## A. 데이터 모델 변경

### A1. Exercise에 보조근 정확도 플래그 추가

**현재 (v4.5)**: 시드 종목과 커스텀 종목 구분이 `is_custom` 필드뿐
**변경 후 (v4.6)**: 커스텀 종목은 사용자가 보조근 매핑을 정확히 모를 수 있으므로 플래그 추가

```typescript
{
  // ... 기존 필드들
  is_custom: boolean;
  muscle_mapping_confidence: 'verified' | 'user_estimated';  // v4.6 신규
}
```

- 시드 종목: `'verified'`
- 커스텀 종목: 기본 `'user_estimated'`
- 사용자가 매핑 검증 완료 표시 시 `'verified'`로 전환 가능 (Phase 2 이후)

**이유**:
- 사용자 추정 매핑은 부위별 볼륨 계산 정확도가 낮음
- Phase 2 추천 엔진에서 이 플래그를 보고 처리 (예: 경고 표시)

## B. 기능 추가/변경

### B1. 같은 세션 내 이전 세트 복사 기능

**배경**: 점진적 과부하는 보통 같은 무게 × 같은 횟수를 여러 세트 반복. 매 세트마다 같은 숫자 입력은 비효율적.

**규칙**:
- 세트 추가 시 기본값은 빈 입력 (기존 동작 유지)
- 별도 버튼("이전 세트 복사" 또는 ⟲ 아이콘)으로 직전 세트 값 복사
- 복사 범위:
  - 본세트 → 본세트: 무게, 횟수, RPE/RIR 모두 복사
  - 워밍업 → 워밍업: 무게, 횟수 복사 (RPE/RIR은 미복사)
  - 워밍업 → 본세트: 복사 버튼 비활성화 또는 무게만 복사
- 직전 세트가 없으면 (첫 세트) 버튼 비활성화

**영향**: F1 운동 기록 입력에 세부 항목 추가

### B2. 커스텀 종목 추가 UI

**배경**: 시드 데이터에 모든 종목이 있을 수 없음. 사용자가 자주 하는 운동이 누락된 경우 매번 JSON 편집은 비현실적.

**입력 필드 (단순화)**:
- 종목 이름 (필수)
- 카테고리 (필수): 컴파운드 / 아이솔레이션
- 기구 (필수): barbell / dumbbell / machine / cable / bodyweight / ez_bar
- 주동근 1개 선택 (필수, 드롭다운에서 Muscle 선택)
- 보조근 0~2개 (선택):
  - 각 보조근에 자극 계수 0.5 또는 0.25 선택
- 권장 휴식 시간 (선택, 카테고리 기반 기본값 자동 제공)
- 권장 반복 범위 (선택, 카테고리 기반 기본값 자동 제공)

**자동 추정/생략 필드**:
- `joint_stress_region_ids`: 기구 + 카테고리 기반 자동 추정
- `movement_patterns`: 자동 추정 또는 비워둠
- `weight_increment_kg`: 카테고리 기반 기본값 (컴파운드 2.5, 아이솔레이션 1.25)
- `fatigue_factor`: 기본 1.0
- `recommendation_bias`: 기본 1.0
- `alternative_group_id`: null (Phase 3에서 사용자가 매칭 또는 자동 추정)
- `skill_level`, `stability`, `unilateral`: 기본값 자동

**위치**: 종목 목록 화면 우상단에 "+" 버튼 → 커스텀 종목 추가 모달

**삭제/수정**:
- 사용자가 추가한 커스텀 종목만 삭제/수정 가능 (`is_custom: true`)
- 시드 종목은 수정 불가
- 사용된 종목 삭제 시도 시: 과거 세션에 영향 안 가도록 soft delete 또는 경고

### B3. 백업 권장 알림

**배경**: 사용자가 백업의 중요성을 인지하지 못하면 데이터 손실 위험.

**규칙**:
- 5회 이상 세션 기록 + 백업 한 번도 안 한 경우, 홈/히스토리 화면 상단에 안내 카드 표시
- "데이터를 안전하게 보관하기 위해 백업을 권장합니다. 5분이면 됩니다."
- 카드 클릭 시 설정 → 백업 화면으로 이동
- 백업 1회 완료 시 안내 카드 사라짐
- 마지막 백업 후 30일 경과 시 다시 표시

## C. UX 원칙 추가

### C1. 색상 시스템 표준화

**배경**: 검은 바탕에 흰 글씨만 있어 버튼 구분이 어려움. 특히 헬스장에서 빠르게 인지해야 하는 액션 버튼이 모두 같은 색이면 사용성 저하.

**원칙**:

**의미와 색을 일치시킨다.**

| 의미 | 색상 카테고리 | Tailwind 권장 |
|------|--------------|---------------|
| 위험 / 종료 / 삭제 | 빨강 계열 | red-400, red-500 |
| 진행 / 추가 / 시작 | 초록 계열 | emerald-400, green-500 |
| 정보 / 기본 / 저장 | 파랑 계열 | sky-400, blue-500 |
| 경고 / 주의 | 노랑/주황 | amber-400, orange-400 |
| 비활성 / 보조 | 회색 | gray-400, slate-400 |

**다크 모드 친화적 채도**:
- 원색 사용 자제. 톤 다운된 색 사용
- Tailwind 기준 `500` 보다 `400` 단계가 다크 배경에 더 부드러움
- 텍스트 가독성: 배경 색상이 있는 버튼은 흰색 또는 거의 흰색 텍스트

**구체 적용 예시 (세션 화면)**:
- "세트 추가" 버튼: 초록 (진행/추가)
- "종목 추가" 버튼: 초록 (진행/추가, 같은 의미라 같은 색)
- "세션 종료" 버튼: 빨강 (종료)
- "삭제" 버튼: 빨강 (위험)
- "저장" 버튼: 파랑 (기본 동작)
- "취소" 버튼: 회색 (보조)

**한 화면 내 색상 가짓수**:
- 한 화면에 너무 많은 색을 쓰면 오히려 혼란
- 권장: 한 화면에 3가지 색 이하 (배경/텍스트 제외)

**영향**: 새 섹션 추가 또는 7.5 체크리스트 항목

## D. 구현 컨벤션

### D1. SPA 라우팅 폴백 설정 필수

**배경**: Vercel 배포 시 `vercel.json` 없으면 루트 외 경로에서 404 발생.

**규칙**:
- 프로젝트 루트에 `vercel.json` 포함
- 내용:
```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```
- 다른 정적 호스팅(Netlify, Cloudflare Pages 등) 사용 시 해당 폴백 설정 추가

**영향**: CLAUDE.md 함정 주의사항에 추가
